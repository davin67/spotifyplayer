In sentry.io, release health uses the following terms --  Crashes, Healthy, Abnormal, Errored, Crashed, Crash Free Users, and Crash Free Sessions -- the definition of which naturally varies according to the platform:

| JavaScript  |        |
|---------|--------|
|Crashes|Number of sessions with a crashed state|
|Crashed|During the session an error with `mechanism.handled===false` occurred calling the `onerror` on `unhandledrejection` handler|
|Crash Free Users|Number of unique users with non-crashed sessions|
|Crash Free Sessions|Number of non-crashed sessions|
|Abnormal|Non applicable for Javascript|
|Healthy|No errors were captured during session life-time|
|Errored| During the session at least one error occurred that did not bubble up to the global handler. The application loading process did not crash|
|Unhandled|An error was captured by the global `onerror` or `onunhandledrejection` handler|

| Node.js  |        |

| Dotnet |        |
|---------|--------|
|Crashes|Number of sessions with a crashed state|
|Crashed|The process was terminated due to an unhandled exception or a request to the server that ended with an error|
|Crash Free Users|Number of unique users with non-crashed sessions|
|Crash Free Sessions|Number of unique sessions that did not crash|
|Abnormal|An unknown session exit. Like due to loss of power or killed by the operating system|
|Healthy|A session without any errors|
|Errored|A crash which experienced errors|
|Unhandled|Not handled by user code|
